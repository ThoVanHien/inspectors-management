# Exam & Certificate Management System - MVP Skill Guide

## 1. Project Context

This system is an internal company web application used to manage:

- Employees who take exams
- Exam creation by Part Leaders
- Online exam taking
- Offline paper exam record uploads
- Certificate management after passing exams
- PDF preview/download for exam results and certificates

The system must be simple, maintainable, and optimized for limited company server resources.

---

## 2. Tech Stack

### Backend

- Node.js
- Express.js
- TypeScript
- dotenv with environment-based configuration
- MySQL
- mysql2 as the database driver
- JWT authentication
- Role-based authorization
- File upload support
- PDF generation with pdfmake

### Frontend

- Angular 17
- Keep Angular NgModule architecture
- Angular Material for UI components
- Route guards for authentication and authorization

### Database

There are two database areas:

1. Company employee database, read-only from this app
2. Application database, owned by this system

The employee database is treated as external master data.

## 3. External Employee Data

The company already has an employee table. The app should not modify this table directly.

Important fields from employee data:

```text
employeeID      - user id, example: a.nguyen
employeeNo      - employee number, example: 123456
subPartCode     - sub part code
partCode        - part code
part            - part name
groupCode       - group code
group           - group name
team            - team name, example: Team A or Team B
status          - 0/1 active or resigned
created         - created date
updated         - updated date
leaderName      - leader name
leaderEmail     - leader email
birthday        - employee birthday
joinDate        - company join date
fullName        - Vietnamese full name
fullNameEn      - English full name
```

Recommended rule:

- Do not store the full employee data again unless needed.
- Store `employee_id` and `employee_no` in the app database.
- Snapshot fields such as `part_code`, `group_code`, `team`, and `full_name` only when needed for reporting or historical records.

## 4. Main Business Rules

### Exam ownership

- A Part Leader creates exams for their own part.
- Employees in that part can take the assigned exams.
- The exam answer key is created by the Part Leader.
- The backend grades employee answers against the answer key.

### Passing rule

Company requirement:

```text
Only 100 points is considered PASS.
Anything below 100 is FAIL.
```

Recommended behavior:

- Save only passed exam attempts as official records.
- Failed attempts do not need to store detailed answers.
- Optionally store a lightweight fail log for statistics or anti-spam tracking.

### Online exam

For online exams:

- Save the employee's real answers only when they pass.
- Store answers in database records, not as PDF or BLOB.
- Use the stored answers to render review UI.
- Generate PDF on demand when the user clicks download.

### Offline exam

For offline paper exams:

- Leader/Admin uploads scanned paper exam PDF.
- Leader/Admin may also upload certificate PDF.
- Store files on disk, not in MySQL BLOB.
- Store only the file path and metadata in the database.

### Certificate

- A certificate is issued only after passing an exam.
- Certificates can be generated by the system or uploaded manually.
- Certificate PDF files should be stored on disk.
- Database stores metadata and file reference.

---

## 5. File Storage Rules

Do not store PDF files as MySQL BLOB.

Use local disk storage first:

```text
/uploads/certificates/YYYY/MM/
/uploads/paper-exams/YYYY/MM/
/uploads/generated-exams/YYYY/MM/   optional
```

The database should only store:

```text
file_path
original_name
stored_name
mime_type
file_size
file_hash
uploaded_by
created_at
```

Recommended optimizations:

- Limit PDF upload size.
- Compress PDFs before storing if possible.
- Use SHA-256 hash to prevent duplicate file storage.
- Split folders by year and month.
- Add soft delete instead of physical delete.
- Use a retention policy for old paper exam scans.

## 7. Tables

## 7.1 users

Stores application login accounts.

```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  employee_id VARCHAR(100) NOT NULL,
  employee_no VARCHAR(50),
  username VARCHAR(100) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  email VARCHAR(255),
  full_name VARCHAR(255),
  part_code VARCHAR(50),
  group_code VARCHAR(50),
  team VARCHAR(100),
  status ENUM('ACTIVE', 'INACTIVE') NOT NULL DEFAULT 'ACTIVE',
  last_login_at DATETIME NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

Notes:

- `employee_id` maps to company `employee.employeeID`.
- `employee_no` maps to company `employee.employeeNo`.
- `part_code`, `group_code`, and `team` may be synced/snapshotted for faster permission checks.

Many-to-many relation between users and roles.

## 7.4 exams

Stores exams created by Part Leaders.

```sql
CREATE TABLE exams (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(50) NOT NULL UNIQUE,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  part_code VARCHAR(50) NOT NULL,
  duration_minutes INT NOT NULL DEFAULT 30,
  pass_score DECIMAL(5,2) NOT NULL DEFAULT 100.00,
  max_attempts INT NULL,
  shuffle_questions BOOLEAN NOT NULL DEFAULT FALSE,
  shuffle_options BOOLEAN NOT NULL DEFAULT FALSE,
  status ENUM('DRAFT', 'PUBLISHED', 'CLOSED', 'ARCHIVED') NOT NULL DEFAULT 'DRAFT',
  created_by BIGINT NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT fk_exams_created_by FOREIGN KEY (created_by) REFERENCES users(id)
);
```

Notes:

- `part_code` controls which part owns the exam.
- `pass_score` is 100 by company rule.
- Keep `pass_score` configurable in case the policy changes later.

## 7.5 questions

Stores questions for exams.

```sql
CREATE TABLE questions (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  exam_id BIGINT NOT NULL,
  type ENUM('SINGLE_CHOICE', 'MULTIPLE_CHOICE', 'TRUE_FALSE') NOT NULL DEFAULT 'SINGLE_CHOICE',
  content TEXT NOT NULL,
  score DECIMAL(5,2) NOT NULL DEFAULT 1.00,
  order_no INT NOT NULL DEFAULT 0,
  status ENUM('ACTIVE', 'INACTIVE') NOT NULL DEFAULT 'ACTIVE',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT fk_questions_exam FOREIGN KEY (exam_id) REFERENCES exams(id)
);
```

MVP simplification:

- Questions belong directly to one exam.
- No separate question bank table yet.
- A question bank can be added later if needed.

## 7.6 question_options

Stores options and answer key.

```sql
CREATE TABLE question_options (

  CONSTRAINT fk_question_options_question FOREIGN KEY (question_id) REFERENCES questions(id)
```

Notes:

## 7.7 exam_attempts

Stores official passed attempts and offline records.

```sql
CREATE TABLE exam_attempts (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  exam_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  employee_id VARCHAR(100) NOT NULL,
  employee_no VARCHAR(50),
  attempt_type ENUM('ONLINE', 'OFFLINE') NOT NULL DEFAULT 'ONLINE',
  status ENUM('PASSED', 'OFFLINE_UPLOADED', 'CANCELLED') NOT NULL DEFAULT 'PASSED',
  total_score DECIMAL(5,2) NOT NULL DEFAULT 100.00,
  is_passed BOOLEAN NOT NULL DEFAULT TRUE,
  started_at DATETIME NULL,
  submitted_at DATETIME NULL,
  paper_exam_file_id BIGINT NULL,
  created_by BIGINT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_exam_attempts_exam FOREIGN KEY (exam_id) REFERENCES exams(id),
  CONSTRAINT fk_exam_attempts_user FOREIGN KEY (user_id) REFERENCES users(id),
  CONSTRAINT fk_exam_attempts_created_by FOREIGN KEY (created_by) REFERENCES users(id)
);
```

Notes:

- For online exams, save only if score is 100.
- For offline exams, `paper_exam_file_id` links to uploaded scanned PDF.
- Since the company only accepts 100-point pass records, `is_passed` is usually true here.
- Keep `total_score` and `is_passed` for reporting and future flexibility.

## 7.8 attempt_answers

Stores real employee answers for passed online attempts.

```sql
CREATE TABLE attempt_answers (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  attempt_id BIGINT NOT NULL,
  question_id BIGINT NOT NULL,
  answer_text TEXT NULL,
  is_correct BOOLEAN NOT NULL DEFAULT TRUE,
  score DECIMAL(5,2) NOT NULL DEFAULT 0.00,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_attempt_answers_attempt FOREIGN KEY (attempt_id) REFERENCES exam_attempts(id),
  CONSTRAINT fk_attempt_answers_question FOREIGN KEY (question_id) REFERENCES questions(id)
);
```

Notes:

- Do not save Leader answers as employee answers.
- Save the actual options selected by the employee.
- Since only passed attempts are saved, `is_correct` is usually true, but keep it for clarity.

## 7.12 employee_certificates

Stores certificates issued to employees.

```sql
CREATE TABLE employee_certificates (

  certificate_id BIGINT NOT NULL,

  certificate_file_id BIGINT NULL,
```

Notes:

- For online exams, `attempt_id` should reference the passed attempt.

## 7.13 files

Stores file metadata only.

```sql
CREATE TABLE files (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  original_name VARCHAR(255) NOT NULL,
  stored_name VARCHAR(255) NOT NULL,
  file_path VARCHAR(500) NOT NULL,
  mime_type VARCHAR(100) NOT NULL,
  file_size BIGINT NOT NULL,
  compressed_size BIGINT NULL,
  file_hash VARCHAR(128) NOT NULL,
  file_type ENUM('CERTIFICATE_PDF', 'PAPER_EXAM_SCAN', 'CERTIFICATE_TEMPLATE', 'OTHER') NOT NULL,
  storage_type ENUM('LOCAL', 'NAS', 'MINIO', 'S3') NOT NULL DEFAULT 'LOCAL',
  status ENUM('ACTIVE', 'DELETED') NOT NULL DEFAULT 'ACTIVE',
  uploaded_by BIGINT NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL,
  UNIQUE KEY uq_file_hash (file_hash),
  CONSTRAINT fk_files_uploaded_by FOREIGN KEY (uploaded_by) REFERENCES users(id)
);
```

Important:

- Do not store file binary data in this table.
- Store the physical file on disk.
- Store metadata and path only.

## 10. Online Exam Flow

```text
Employee opens assigned exam
↓
Frontend loads questions without exposing is_correct
↓
Employee submits answers
↓
Backend loads answer key from question_options
↓
Backend grades submission
↓
If score < 100:
    return FAIL result to frontend
    do not save official attempt answers
    optionally save exam_fail_logs only
↓
If score = 100:
    create exam_attempts record
    create attempt_answers records
    create attempt_answer_options records
    create employee_certificates record if exam has certificate
    optionally generate certificate PDF and save file metadata
```

Important security rule:

- Never send `is_correct` to frontend during exam taking.
- Grade only on the backend.

## 18. Important Implementation Notes

### Do not trust frontend grading

Frontend can validate basic required fields, but final grading must happen on backend.

### Do not expose correct answers during exam

When loading exam questions for employees, remove:

```text
is_correct
score details if unnecessary
internal metadata
```

### Do not store failed detailed answers unless required

Because the company only recognizes 100-point pass attempts, store official records only for passed attempts.

### Store actual employee answers

Do not copy Part Leader answer key as employee answers.

For passed online attempts, store what the employee actually selected.

### PDF strategy

- Online exam PDF: generate on demand with pdfmake.
- Certificate PDF: generate and store if it is an official document.
- Offline paper exam PDF: upload and store file path.

### File strategy

- Never use MySQL BLOB for PDFs.
- Use local disk first.
- Store metadata in `files`.
- Use `file_hash` to avoid duplicate physical storage.
